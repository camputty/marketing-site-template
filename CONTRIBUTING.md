# Contributing

Use a focused branch and keep changes within the starter’s fixed architecture. New baseline dependencies or infrastructure must remove meaningful repeated work without imposing project-specific product decisions.

Before opening a pull request:

```bash
pnpm check
pnpm test:e2e
```

Do not add public sample content, generic marketing sections, fabricated proof, or credentials. Explain any content-contract change and include migration guidance when it can affect repositories created from an earlier template version.

Contributions are accepted under the repository's MIT License.
